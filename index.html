KPIs Formulas Used

Churn Rate:

Calculates the ratio of churned customers to the total number of customers, using the CALCULATE function with COUNTROWS and FILTER to count the rows in the 'Churn' table where the churn status is "Yes."

Average Customer Lifespan:

Calculates the average duration of the customer relationship for churned customers, using the DIVIDE function with CALCULATE, SUM, and FILTER to calculate the sum of 'tenure' for churned customers and then divide it by the total number of customers.

Customer Lifetime Value (CLV):

Calculates the customer lifetime value by dividing the total revenue by the total number of customers and multiplying it by the average customer lifespan. It uses CALCULATE, SUM, FILTER, and DIVIDE functions to calculate the sum of 'tenure' for churned customers, divide it by the total number of customers, and then multiply it by the total revenue.

Monthly Recurring Revenue

Calculates the Monthly Recurring Revenue (MRR) by summing the 'MonthlyCharges' column for each row in the 'Churn' table, using the SUMX function. It iterates over each row in the table and adds up the monthly charges for all customers.

Non Churn Cust:

Calculates the count of non-churn customers using the CALCULATE function with FILTER to count the rows in the 'Churn' table where the churn status is "No."

Active Months:

Calculates the total number of months that non-churn customers have been active using the CALCULATE function with SUM and FILTER to calculate the sum of 'tenure' for non-churn customers.

Active Customers:

Calculates the count of active customers using the COUNTROWS function with FILTER to count the rows in the 'Churn' table where the churn status is "No."

Churn Cust:

Calculates the count of churned customers using the CALCULATE function with FILTER to count the rows in the 'Churn' table where the churn status is "Yes."

Customer Retention Rate:

Calculates the customer retention rate by subtracting the count of lost customers from the total number of customers and then dividing it by the total number of customers.

Data Cleaning Workflow

  1. Data Assessment: Start by thoroughly understanding the dataset and its structure. Identify the variables, data types, and any potential data quality issues or inconsistencies.

  2. Data Profiling: Perform initial data profiling to gain insights into the dataset. This includes assessing data distributions, checking for missing values, outliers, and identifying potential data quality issues.

  3. Standardize Data Formats: Standardize data formats across different fields to ensure consistency and facilitate seamless integration. This may involve converting date formats, normalizing text values, or standardizing categorical variables.

  4. Handle Missing Values: Implement data imputation techniques to handle missing values. This can be done using methods such as mean, median, or regression imputation based on the characteristics of the variables and the data.

  5. Cleanse Textual Data: Cleanse textual data by removing special characters, correcting typos, and standardizing text formats. This can involve techniques like removing punctuation, applying case normalization, or using regular expressions for pattern matching and text manipulation.

  6. Remove Irrelevant or Redundant Variables: Identify and remove variables/columns that do not contribute to the analysis or contain duplicate or redundant information. This helps streamline the dataset and improves data processing efficiency.

  7. Address Data Outliers and Anomalies: Handle data outliers and anomalies through techniques such as data smoothing, outlier detection algorithms, or appropriate data transformation methods. This helps ensure that extreme values do not skew the analysis or modeling results.

  8. Ensure Data Consistency and Integrity: Identify and resolve inconsistent or conflicting data values to maintain data consistency and integrity throughout the dataset. This can involve techniques such as data validation rules or resolving discrepancies between related variables.

  9. Validate Data Quality and Structure: Perform data validation and verification to ensure that the cleaned data meets defined quality standards and aligns with the expected data structure. This includes checking data ranges, constraints, and validating against predefined rules or business requirements.

  10. Document Data Cleaning Processes: Document the data cleaning processes and steps taken to maintain transparency and reproducibility. This includes documenting the techniques, transformations, and validations applied, as well as any decisions made during the cleaning process.
